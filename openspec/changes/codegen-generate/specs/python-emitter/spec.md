# Spec: python-emitter

## Purpose

`codegen/python_emitter.py` converts a list of internal tool definitions (parsed from an OpenAPI spec) into a complete, self-contained `main.py` source string for a Databricks App MCP server. No file I/O — input is a list of dicts, output is a string.

---

## Public Interface

```python
def emit_module(
    tools: list[dict],
    connection_name: str,
    service_name: str,
    spec_source: str,
) -> str:
    """Return the full main.py source as a string."""
```

All other functions in the module are private helpers (`_` prefix) and tested directly.

---

## Requirements

### R1: Type mapping

`_map_type(openapi_type: str | None) -> str` maps OpenAPI primitive types to Python type hint strings.

| Input | Output |
|-------|--------|
| `"string"` | `"str"` |
| `"integer"` | `"int"` |
| `"number"` | `"float"` |
| `"boolean"` | `"bool"` |
| `"array"` | `"list"` |
| `"object"` | `"dict"` |
| `None` / unknown | `"dict"` |

### R2: Function signature generation

`_emit_signature(tool: dict) -> str` returns the `async def` line for a tool.

- Required params (including all path params) appear first, no default
- Optional params appear after, with `= None` default (except `bool` → `= False`, `int`/`float` → `= 0`)
- Return type is always `-> str`
- Example: `async def get_item(item_id: str, include_deleted: bool = False) -> str:`

### R3: Path interpolation

`_emit_path(path: str, path_params: list[str]) -> str` returns a Python expression for the URL path.

- If `path_params` is empty: returns the path as a plain string literal `"/items"`
- If `path_params` is non-empty: returns an f-string `f"/items/{item_id}"`
- Braces in the path template (`{item_id}`) map directly to f-string interpolation variables

### R4: Request call emission

`_emit_request_call(tool: dict) -> str` returns the `return _uc_request(...)` line for a tool handler.

- First arg: HTTP method string literal (`"GET"`, `"POST"`, etc.)
- Second arg: path expression from `_emit_path`
- If query params exist: `query_params={...}` keyword arg with param names as keys and variable names as values
- If body params exist and method is not GET: `body={...}` keyword arg
- Path params are NOT included in query_params or body

### R5: Full tool function emission

`_emit_tool_function(tool: dict) -> str` returns the complete decorated function source for one tool:

```python
@mcp.tool()
async def list_items(limit: int = 0, offset: int = 0) -> str:
    """List all items."""
    return _uc_request("GET", "/items", query_params={"limit": limit, "offset": offset})
```

- Docstring is the tool description (single line, double-quoted)
- If description is empty or None, omit the docstring
- Blank line after each function

### R6: Module header

`_emit_header(service_name: str, connection_name: str, spec_source: str) -> str` returns the top of the file:

```python
# Generated by uc-mcp generate
# Spec: <spec_source>
# Connection: <connection_name>

from __future__ import annotations

import json
from collections.abc import AsyncIterator
from contextlib import asynccontextmanager

from databricks.sdk import WorkspaceClient
from databricks.sdk.config import Config
from mcp.server.fastmcp import FastMCP
from starlette.applications import Starlette
from starlette.middleware.base import BaseHTTPMiddleware
from starlette.requests import Request
from starlette.routing import Mount
import uvicorn

mcp = FastMCP("<service_name>")
CONNECTION_NAME = "<connection_name>"
```

### R7: `_uc_request` helper emission

`_emit_uc_request_helper() -> str` returns the shared HTTP proxy helper inlined in the module:

```python
def _uc_request(
    method: str,
    path: str,
    *,
    body: dict | None = None,
    query_params: dict | None = None,
    _client: WorkspaceClient | None = None,
) -> str:
    client = _client or WorkspaceClient()
    from databricks.sdk.service.serving import ExternalFunctionRequestHttpMethod
    _method = getattr(ExternalFunctionRequestHttpMethod, method)
    resp = client.serving_endpoints.http_request(
        conn=CONNECTION_NAME,
        method=_method,
        path=path,
        headers={},
        json=body or {},
        params=query_params or {},
    )
    return resp.text
```

### R8: Per-user auth middleware emission

`_emit_middleware() -> str` returns the `ForwardedTokenMiddleware` class and the Starlette app setup:

- Middleware extracts `X-Forwarded-Access-Token` header and stores it in request state
- Tool handlers use per-request `WorkspaceClient(token=...)` if token present, else default client
- Exposes `/mcp` route via `StreamableHTTPSessionManager`
- `uvicorn.run(app, host="0.0.0.0", port=8000)` in `if __name__ == "__main__"`

### R9: Full module assembly

`emit_module(tools, connection_name, service_name, spec_source) -> str` concatenates:

1. Header (`_emit_header`)
2. `_uc_request` helper (`_emit_uc_request_helper`)
3. Middleware + app setup (`_emit_middleware`)
4. One `_emit_tool_function` per tool, in input order
5. `if __name__ == "__main__": uvicorn.run(...)` block

The result must be valid Python (parseable by `ast.parse`).

### R10: Empty tool list

`emit_module` with an empty `tools` list returns a valid module with no tool functions — just the header, helper, middleware, and entrypoint.

### R11: Tool name sanitisation

If a tool name contains characters invalid in a Python identifier (e.g. `-`, spaces), `_emit_tool_function` replaces them with `_`. Tool names from `from_openapi.py` are already snake_case, so this is a safety net only.

## Scenarios

| # | Input | Expected output |
|---|-------|-----------------|
| S1 | `_map_type("string")` | `"str"` |
| S2 | `_map_type(None)` | `"dict"` |
| S3 | `_map_type("unknown_future_type")` | `"dict"` |
| S4 | Signature with required path param + optional query param | required first, optional with default |
| S5 | GET tool with query params | `_uc_request("GET", "/x", query_params={...})` |
| S6 | POST tool with body params | `_uc_request("POST", "/x", body={...})` |
| S7 | Path with two placeholders | f-string with both vars |
| S8 | Tool with no description | function emitted without docstring |
| S9 | `emit_module` output passes `ast.parse` | no SyntaxError |
| S10 | `emit_module` with empty tools | valid module, no tool functions |
